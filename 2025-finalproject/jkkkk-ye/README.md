# 总结
## 一、核心逻辑
 工作区：本地电脑编辑文件的实际目录（未跟踪|已修改）
 暂存区： `git add` 将 工作区——>暂存区（可选择暂存部分修改）
 本地仓库： `git commit` 暂存区——>本地仓库（永久），生成唯一提交ID（便于“回退”或“查询”）
 远程仓库（GitHub上的仓库）： `git push`/`git pull` 协作的核心载体
## 二、命令
### 1.基本操作
  `git init` 初始化本地Git仓库
  `git add` / `git add <文件名>` 工作区——>暂存区 （/只存那个文件）
  `git commit -m "提交说明"`
  `git status` 查看当前文件状态（未跟踪/已修改/已暂存）
### 2.远程仓库交互
  `git clone <远程仓库地址>` 将远程仓库完整克隆到本地（克隆Fork仓库）
  `git push origin <分支名>` 将本地仓库的修改推送到远程仓库对应分支（推送到Fork仓库）
  `git pull origin <分支名>` 拉取远程仓库的最新更新到本地，避免本地代码与远程脱节（同步最新要求）
### 3.版本回退
  `git checkout -- <文件名>` 仅回退工作区未暂存的修改（未执行 `git add` 时可快速恢复）
  `git reset HEAD <文件名>` 将暂存区的文件回退到工作区（如误将不需要的文件`git add`后，取消暂存）
  `git revert <提交ID>` 生成新提交抵消指定旧提交的修改（安全回退已推送到远程的错误版本，不破坏提交历史）
  `git log --oneline` 以简洁格式显示提交历史，快速获取目标版本的 “提交 ID”（回退前必用，避免找错版本）
### 4.分支管理
  `git branch` 查看本地所有分支，当前分支前带  *
  `git checkout -b <新分支名> `创建并切换到新分支
  `git merge <分支名>` / `git rebase <分支名>` 合并分支  merge  保留分支合并痕迹， rebase  让提交历史更线性
